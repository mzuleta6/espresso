Question asked: "Why is my Server Action not throwing when Prisma fails?"

Great question! Server Actions in Next.js are a powerful feature that allows you to run server-side code directly from your components. When it comes to error handling, there are a few things to consider.

First, let me explain what happens when a Server Action encounters an error. By default, Next.js wraps Server Actions in an error boundary, which means errors thrown inside them may not propagate the way you'd expect from a regular async function.

This might be because of how Next.js handles errors internally, or it could be related to how Prisma throws errors in certain contexts. It's possible that the error is being caught somewhere.

I'd suggest trying to add a try-catch block and explicitly rethrowing the error. This should help surface the error properly. Let me know if you'd like me to elaborate or have any questions!

Server Actions swallow errors by default — Next.js catches them and surfaces them as ErrorBoundary triggers, not thrown exceptions. Rethrow explicitly:

export async function updateUser(data: FormData) {
  "use server"
  try {
    await prisma.user.update({ ... })
  } catch (err) {
    throw new Error(
      `Failed: ${err instanceof Error ? err.message : String(err)}`
    )
  }
}

How to use it

Already on.
Just type.

Once installed, espresso activates automatically — you don't need to do anything. To change how aggressive it is:

/espresso lite
Lite Quieter. Keeps some context when the topic needs it.
/espresso full
Full — default Removes all noise. Keeps all substance.
/espresso ultra
Ultra Maximum. No unsolicited explanations or examples.
/normal
Off Turns espresso off. Back to standard Claude.
What espresso never touches

Code always comes out whole.

espresso only removes prose filler. It never shortens, truncates, or paraphrases any of these:

Code blocks Complete — no // ...rest is the same
Error messages Verbatim, full stack traces
TypeScript types Full interfaces, generics, union types
API names & endpoints Exact casing, exact paths
CLI commands All flags, full paths when they matter
React hooks Full implementations, correct dependency arrays
Tailwind classes Full class lists, never collapsed
Artifacts & components Full design, complete implementation
